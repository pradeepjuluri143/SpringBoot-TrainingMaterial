üîß Constructors

A Constructor is like a factory worker who sets up your object when it's created. It gives initial values to your object!

üéÇ Birthday Cake Example

Constructor = Cake Decorator

When you order a cake, the decorator asks: "What flavor? What color? How many candles?"

The constructor does the same - it sets up your object with initial values!

class Student { String name; int age; // Constructor - runs when object is created Student(String studentName, int studentAge) { name = studentName; age = studentAge; System.out.println("New student created: " + name); } } // Using constructor Student john = new Student("John", 20); Student mary = new Student("Mary", 19);

üîí Encapsulation

Encapsulation is like putting your valuables in a safe box. You hide the important stuff and only allow access through special methods!

üè¶ Bank Account Example

Private = Your money in the bank vault (hidden from everyone)

Public methods = Bank teller (the only way to access your money)

You can't directly grab money from the vault - you must use the teller (methods)!

class BankAccount { private double balance; // Hidden from outside // Public method to deposit money public void deposit(double amount) { if(amount > 0) { balance += amount; System.out.println("Deposited: $" + amount); } } // Public method to check balance public double getBalance() { return balance; } }

Key Points to Remember:

  • Private = Hidden from outside world
  • Public = Can be accessed by anyone
  • Use getter/setter methods to access private data
  • Protects data from unauthorized access

‚ö° Static vs Non-Static

Static belongs to the class (shared by all). Non-static belongs to individual objects!

üè´ School Example

Static = School Name (same for all students)

Non-static = Student's Name (different for each student)

All students share the same school name, but each has their own name!

class Student { static String schoolName = "ABC School"; // Shared by all String studentName; // Individual to each static void showSchoolInfo() { // Static method System.out.println("School: " + schoolName); } void showStudentInfo() { // Non-static method System.out.println("Student: " + studentName); } } // Usage Student.showSchoolInfo(); // Called on class Student s1 = new Student(); s1.showStudentInfo(); // Called on object

üéÆ School Demo!

Easy Way to Remember:
Static = Statue üóø (doesn't move, same for all)
Non-static = People üö∂ (each person is different)

Key Points to Remember:

  • Static belongs to class, not objects
  • Static can be accessed without creating objects
  • Non-static belongs to individual objects
  • Static methods can't access non-static variables directly

üë®‚Äçüë©‚Äçüëß‚Äçüë¶ Inheritance

Inheritance is like getting traits from your parents. Child classes inherit properties and methods from parent classes!

üêï Animal Family Example

Parent = Animal (has basic features like eat, sleep)

Child = Dog (inherits eat, sleep + adds bark)

Child = Cat (inherits eat, sleep + adds meow)

All animals eat and sleep, but each has special abilities!

// Parent class class Animal { void eat() { System.out.println("Animal is eating"); } void sleep() { System.out.println("Animal is sleeping"); } } // Child class inherits from Animal class Dog extends Animal { void bark() { System.out.println("Dog is barking: Woof!"); } } // Usage Dog myDog = new Dog(); myDog.eat(); // Inherited from Animal myDog.bark(); // Own method

  • Child can add its own methods and properties
  • Promotes code reusability

üé® Abstraction

Abstraction means showing only what's necessary and hiding complex details. Like using a TV remote without knowing how TV works!

üöó Car Driving Example

What you see: Steering wheel, gas pedal, brake pedal

What's hidden: Engine mechanics, fuel injection, transmission

You can drive without knowing how the engine works internally!

// Abstract class - cannot create objects directly abstract class Vehicle { abstract void start(); // Must be implemented by child // Normal method - can be used by all vehicles void stop() { System.out.println("Vehicle stopped"); } } class Car extends Vehicle { void start() { System.out.println("Car started with key üîë"); } } class Bike extends Vehicle { void start() { System.out.println("Bike started with kick ‚ö°"); } }

Key Points to Remember:

  • Hides complex implementation details
  • Shows only essential features
  • Abstract classes cannot be instantiated
  • Child classes must implement abstract methods

üîå Interfaces

Interface is like a contract or agreement. It says "You MUST have these methods" but doesn't tell you HOW to implement them!

üîå Electronic Device Example

Interface = Power Plug (standard shape)

Devices = TV, Laptop, Phone (all different, but all can plug in)

The plug shape is the same (interface), but each device works differently inside!

// Interface - like a contract interface Flyable { void fly(); // All flyable things must implement this void land(); // All flyable things must implement this } class Bird implements Flyable { public void fly() { System.out.println("Bird flaps wings and flies üê¶"); } public void land() { System.out.println("Bird lands on tree branch"); } } class Airplane implements Flyable { public void fly() { System.out.println("Airplane uses engines to fly ‚úàÔ∏è"); } public void land() { System.out.println("Airplane lands on runway"); } }

Easy Way to Remember:
Interface = Job Description üìã
Lists what you must do, but you decide HOW to do it!

Key Points to Remember:

  • Interface is a contract - must implement all methods
  • Cannot create objects of interface directly
  • Class can implement multiple interfaces
  • All methods in interface are public by default
